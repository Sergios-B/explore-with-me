# Explore With Me (Дипломный проект)

Cервис-платформа для организации мероприятий и поиска компании для участия в них. Приложение позволяет пользователям публиковать информацию о событиях, регистрироваться на мероприятия, а администраторам — модерировать контент и управлять системой.

## 🛠️ Технологический стек
* **Java 17 / Spring Boot 3**
* **Spring Data JPA / PostgreSQL** (две независимые базы данных)
* **Docker & Docker Compose** (контейнеризация всех сервисов и БД)
* **Maven** (многомодульная сборка)
* **Lombok / MapStruct / Postman**

## 📐 Архитектура проекта
Проект разделен на два ключевых сервиса, работающих в изолированных контейнерах:
1. **Основной сервис (`ewm-main-service`)** — отвечает за всю бизнес-логику: работу с пользователями, категориями, подборками, запросами на участие и мероприятиями.
2. **Сервис статистики (`ewm-stats-service`)** — собирает данные о просмотрах страниц и позволяет выгружать аналитику (состоит из HTTP-сервера, DTO-модуля и HTTP-клиента).

## ✨ Дополнительная функциональность: Подписки (`subscriptions`)
В рамках финального этапа реализована социальная механика **подписок на организаторов**, которая включает:
* **Private API**: Возможность подписаться на интересного организатора мероприятий или отменить подписку.
* **Валидация бизнес-правил**: Защита от повторных подписок на одного человека и строгий запрет на подписку на самого себя (с генерацией статуса `409 Conflict`).
* **Персональная лента событий (`feed`)**: Индивидуальный эндпоинт, который выгружает список только актуальных и опубликованных мероприятий от тех авторов, на которых подписан пользователь, с автосортировкой по дате начала.
* **Оптимизация БД**: Добавлены составные уникальные ключи и индексы для исключения лишней нагрузки на PostgreSQL при работе со списками.

## 🚀 Запуск приложения через Docker
Для локального развертывания всей инфраструктуры приложения (2 сервиса + 2 базы данных) выполните в корне проекта команду:

```bash
docker-compose up -d --build
```

## 📬 Тестирование
Базовые сценарии проверены автотестами платформы. Тестирование новой фичи подписок и ленты находится в папке:
* `postman/feature.json` — импортируйте эту коллекцию в Postman для ручного или автоматического прогона тестов.
